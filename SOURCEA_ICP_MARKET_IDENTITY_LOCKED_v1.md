funds, algorithmic asset managers, quant desks inside tier-1 banks.

cryptographically-receipted, tamper-evident proof of rule compliance
at HFT execution speed.

SourceA closes that gap.

to satisfy a regulatory examination. Receipt logged. Validator HARD FAIL

### VERTICAL 2 — AUTONOMOUS SOFTWARE ENGINEERING ENGINE
### (Repository-Scale Refactoring)

#### What it is
An autonomous code transformation engine that operates at repository scale —
capable of systematic refactoring, migration, and restructuring of codebases
too large for manual intervention — with a deterministic governance layer
that requires human approval before any production merge.

#### The market
Large enterprises with legacy codebases that are economically impossible
to refactor manually. Banks with 30-year-old COBOL. Telecoms with 10M+
line monorepos. Insurance carriers that cannot migrate because the
institutional knowledge of the codebase has been lost. Government agencies
with critical infrastructure written in end-of-life languages.

This is not a developer assistant (Cursor / Copilot class operates file-by-file).
SourceA's autonomous software engineering engine operates at the repository level —
understanding the full dependency graph, executing systematic transformations,
and routing every proposed change through a governance checkpoint before it
touches production.

#### Buyer pain (exact)
Manual refactoring of a 5M+ line codebase takes years and costs tens of millions.
AI-assisted refactoring at file level (current state of the art) does not scale
to the problem. Autonomous refactoring at repository scale exists nowhere.

The blocker to adopting autonomous code transformation is governance:
boards and engineering leadership will not approve an autonomous system
that changes production code without human visibility, approval checkpoints,
and rollback receipts.

SourceA's write-path validator, receipt chain, and HARD FAIL on tamper
is the governance layer that makes autonomous repository transformation
safe enough to approve.

#### The W2 connection
W2 (one write path, receipt, validator HARD FAIL) is the proof-of-concept
for this vertical. The demo is: autonomous change executed, receipt logged,
validator confirms no tampering, human approval checkpoint triggered before merge.
That is the exact artifact an enterprise engineering board needs to approve
autonomous codebase transformation.

| Org type | Financial institution · Telecom · Insurance carrier · Government agency · Large enterprise with legacy debt |
| Geography | Canada primary · US · UK · EU |
| Company size | 1,000–100,000 employees |
| Budget signal | Multi-year transformation contract · infrastructure licensing |
| Deal size | $100K–$5M+ engagement |
| Decision maker | CTO + CFO jointly · Board technology committee for largest deals |
| Sales cycle | 6–18 months · POC-gated · legal review required |

#### Entry motion
1. W1+W2 combined demo: show autonomous refactoring of a sample repo.
   Show write path. Show receipt. Show validator HARD FAIL on simulated tamper.
   Show human approval checkpoint. That is the enterprise engineering pitch.
2. Pilot engagement: run SourceA engine against a bounded subset of their

for production code changes.
No existing vendor provides autonomous repository-scale transformation
