## Core principle

**No single layer owns all truth.** Each layer owns a different truth domain.

They are co-equal — not hierarchical. Connected by alignment artifacts and binding contract, not dominance.

```
  DESIGN (SSOT)     EXECUTION (Runtime)     DELIVERY (Repo)
         \                 |                    /
          \                |                   /
           -------- Alignment layer ----------
```

**Key line:** SSOT describes · Runtime proves · Repo ships.

---

## Layer 1 — DESIGN (SSOT)

**Owns:** intent, boundaries, naming, definitions, ecosystem structure, long-term target architecture.

**Question:** What should exist?

**Does NOT own:** running processes, deployments, runtime state, product releases.

---

## Layer 2 — EXECUTION (Runtime)

**Owns:** running services, active agents, live ports, databases, queues, infrastructure state, actual behavior.

**Question:** What actually exists right now?

**Does NOT own:** ecosystem definitions, product positioning, registry structure.

**Rule:** Runtime is authoritative for reality. Divergence from SSOT ideal is not automatically debt.

---

## Layer 3 — DELIVERY (Repo)

**Owns:** product code, CI/CD, releases, features, public documentation, product APIs.

**Question:** What is being shipped?

**Does NOT own:** ecosystem architecture, SSOT definitions, runtime observations.

**Rule:** Noetfield GitHub may ship without waiting for mono registry or `ANNOUNCEMENT_BOARD`.

## Conflict resolution rules

### Rule 1 — SSOT vs Runtime

Do not assume Runtime is wrong. Create **Drift Record** (status, owner).

**Example:** SSOT target = one spine; Runtime = four services → Drift | Accepted | Owner: Runtime.

### Rule 2 — Repo vs SSOT

Boundary violated → **Conflict (Type C)**. Boundaries preserved → **Evolution** (not conflict).

### Rule 3 — Runtime vs Repo

Runtime wins reality. Repo wins intent-to-ship. Code not deployed ≠ contradiction.

### Rule 4 — Criticality

- **Critical:** submodule collapse, payment routing on public surface, `PRODUCT_TRUTH` breach.
- **Non-critical:** ports, extra services, PM2 names, temporary topology drift.
