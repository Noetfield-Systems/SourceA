## 11. Document originality — how to find THE source among hundreds

**“Original” does not mean:** oldest · newest · longest · the pack map · the prompt · chat · hub copy.

**“Original” means:** the **one canonical active LOCKED doc for that topic**, registered at SourceA root, listed in the authority index.

### 11.1 The five-rank truth ladder (whole system)

From `SINA_SOURCE_ALIGNMENT_LAW_LOCKED_v1.md` §2 — **never invert:**

| Rank | What | Original? |
|------|------|-----------|
| **1** | Latest `*_LOCKED_vN.md` at **SourceA root** for the topic | **YES — SOURCE** |
| **2** | Hub / `command-data.json` / machine mirror cited in index row | Live **projection** of rank 1 |
| **3** | `archive/attachments/` | Staging — not source |
| **4** | `archive/attachments/examples/` | Examples — not source |
| **5** | Chat · advisor paste · RESEARCH yaml | **Input only** |

### 11.2 Agent algorithm (doc vs doc — 8 steps)

When two or more docs seem to say different things:

```text
1. TOPIC     — What subject? (session process · closeout · WTM · factory · …)
2. INDEX     — Open SINA_AUTHORITY_INDEX_MAP_LOCKED_v1.md → row for topic
3. CANONICAL — “Canonical doc (root)” column = the original for that topic
4. VERSION   — Same filename family? Highest vN LOCKED at root wins
5. ARCHIVE   — Path under archive/superseded/? → DEAD — never cite as active
6. REGISTER  — No index row? → NOT canonical until GOV_UNIFY intake + row added
7. PACKAGE   — Batch 2 · Cross-doc · prompt · Canvas? → POINTER ONLY — never original
8. OVERRIDE  — Index “Overrides / wins over” column beats informal prose elsewhere
```

**If step 2 finds no row:** use `SINA_GOVERNANCE_ENTRY_LOCKED_v1.md` §0 branch — still do **not** pick the newest chat or longest markdown.

## 12. Plain definitions — topic · row · path · law · executive effect

### 12.1 Vocabulary (no jargon)

| Word | Plain meaning | Example |
|------|---------------|---------|
| **Topic** | The **subject** one doc owns | “daily session process” · “closeout format” · “WTM steps” |
| **Authority index row** | **One line** in `SINA_AUTHORITY_INDEX_MAP_LOCKED_v1.md` for that topic | Row `FIVE_STEP_APEX` |
| **Path** | **File location logged** | `~/Desktop/SourceA/SOURCEA_FIVE_STEP_AUTONOMOUS_PROGRESS_BLUEPRINT_LOCKED_v1.md` |
| **Law** | **Binding rule** agents must obey for that topic (LOCKED, registered) | Contents of that file for its topic only |
| **Registry** | Machine lookup table — not the law text itself | Authority index · doc library · sequence registry |

### 12.2 What “the path is law” means — executive effect

**Sentence:** For topic T, the path in the authority index row **is** the law.

**Real effect when machine obeys:**

1. Agent gets task touching topic T  
2. Agent opens **that path** — not chat · not hub alone · not similar filename  
3. Agent **acts** per that file for topic T  
4. Agent **does not** invent rule from pack map, prompt, or newer draft  
5. If two paths disagree on **same** topic → **index row wins** · other file must be wrong, superseded, or pointer-only  

**What it does NOT mean:** every file present is law · the folder is law · the pack is one law · newest file wins.

### 12.3 Three different “priority” systems — do not mix

You are right: reality has **many layers**. These are **different** P0/P1/tier words:

| System | Question it answers | Example | Wins conflicts? |
|--------|---------------------|---------|-----------------|
| **A — Authority index** | Which **file is law** for topic T? | `FIVE_STEP_APEX` row | **YES** (for that topic) |
| **B — Doc library P0/P1/P2** | How **soon must agent read** this doc? | `important_docs_index.py` | **NO** — read urgency only |
| **C — Sequence registry** | What was **written when**? | `SOURCE_A_DOCUMENT_SEQUENCE_REGISTRY` | **NO** — chronology only |
| **D — Storage tier 1/2/3** | Where must file **live**? | `FILE_STORAGE_GOVERNANCE` | **NO** — location rule |
| **E — SSOT ecosystem stack** | Who wins **structure** disputes? | `SINA_OS_SSOT` → L0-law | **YES** (ecosystem shape) |
| **F — Pack internal slot** | Which pack doc for **which sub-job**? | Batch 2 §INTEGRITY PACK 5 | **NO** — routes inside pack only |
| **G — Founder P0 goal** | What matters **this week**? | `GOAL_HIERARCHY` · hub Today | **YES** for priority work · not which doc is law |

**Machine rule:** Resolve **topic → authority row (A)** first. Use **B** for read order. Never use **C** (newest) as law. Use **F** only after topic known.
