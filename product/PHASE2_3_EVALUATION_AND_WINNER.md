## Winner 1 — MergePack

### 1. Product Definition

| | |
|--|--|
| **Name** | MergePack |
| **One-line** | Merge PDFs in seconds with page control — no Adobe, no account required for first merge. |
| **Target user** | Office admin, freelancer, realtor, student, ops person who gets 3–20 PDFs and needs one file today. |

### 2. Why It Wins

- **vs others:** Zero LLM cost at MVP; deterministic output; instant “aha.”
- **Why now:** PDF volume up (e-sign, scans, AI exports); incumbents are heavy (Acrobat) or ad-spam sites.
- **Money:** Pay-per-merge or $9/mo before user hits 10th merge — decision in 60 seconds.

### 3. MVP Architecture

| Layer | Choice |
|-------|--------|
| Backend | Python 3.12 + **FastAPI** |
| PDF | **pypdf** (merge, reorder, optional page delete) |
| Frontend | **Next.js** single page or static HTML + htmx |
| AI | **None v1** (optional v2: OCR titles) |
| DB | **None v1** (stateless); SQLite later for usage limits |
| Integrations | **Stripe Checkout** (one-time + sub); optional **S3** for 24h file retention |

### 4. Build Plan (7 days)

| Days | Work |
|------|------|
| **1–2** | FastAPI `POST /merge`, multipart upload, pypdf pipeline, 50MB limit, return PDF stream |
| **3–4** | UI: dropzone, drag reorder thumbnails (page count only), download; error states |
| **5–6** | Stripe: 3 free merges/day by IP cookie + paid unlimited; basic analytics |
| **7** | Deploy API **Railway**, UI **Vercel**, domain, smoke test, Product Hunt draft |

### 5. Monetization

| | |
|--|--|
| **Model** | Freemium: 3 merges/day free · **$9/mo** unlimited · **$2** one-off day pass |
| **First payer** | Post in r/smallbusiness, realtor Facebook groups — “merged 12 disclosure PDFs in one click” |
| **Hook** | Watermark on free only if needed; speed + no signup for first merge |

### 6. Distribution

| | |
|--|--|
| **7 days** | SEO page “merge pdf no signup”, Twitter/X demo GIF, Indie Hackers launch |
| **Viral** | Output file small → users email it; footer “Merged with MergePack” optional on free tier |
| **PLG** | First merge no login; login only for history (v2) |

## Winner 2 — FormToPDF

### 1. Product Definition

| | |
|--|--|
| **Name** | FormToPDF |
| **One-line** | Turn form answers into a branded invoice or quote PDF in one click. |
| **Target user** | Freelancer, contractor, micro-agency sending quotes weekly. |

| 3–4 | UI form + logo upload + preview |
| 5–6 | Stripe $15/mo, save last 5 PDFs to disk |
| 7 | Deploy + post in freelancer communities |

### 5. Monetization

$15/mo unlimited PDFs · $3 per PDF pay-as-you-go · First payer: your network + Upwork-style subs

### 6. Distribution

SEO “invoice generator pdf”, TikTok “quote in 30 sec”, template gallery share

# PHASE 3 — Primary product: **MergePack**

**Why not FormToPDF/CSVDoctor first:** MergePack has lowest integration risk, highest MVP speed score, clearest SEO demand (“merge pdf”), and no HIPAA/template/LLM variables.

### MVP scope (cut list)

| In v1 | Cut v2+ |
|-------|---------|
| Merge 2–30 PDFs | OCR, Word, images |
| Reorder files (not per-page v1) | Per-page drag UI |
| Download merged PDF | Cloud storage account |
| 3 free / day, Stripe unlimited | Teams, API keys |
| | Redaction, sign |

### API endpoints (v1)

| Method | Path | Purpose |
|--------|------|---------|
| GET | `/health` | Liveness |
| POST | `/v1/merge` | multipart `files[]` → `application/pdf` |
| GET | `/v1/limits` | `{ remaining_today, tier }` |
| POST | `/v1/checkout` | Stripe session URL (day pass / sub) |
| POST | `/v1/webhooks/stripe` | Activate paid tier |

### Database schema (v1 optional — SQLite)

```sql
-- only if persisting usage across devices
CREATE TABLE usage_daily (
  id INTEGER PRIMARY KEY,
  ip_hash TEXT NOT NULL,
  day TEXT NOT NULL,
  merge_count INTEGER DEFAULT 0,
  UNIQUE(ip_hash, day)
);

CREATE TABLE subscription (
  stripe_customer_id TEXT PRIMARY KEY,
  email TEXT,
  active_until TEXT
);
```

**v1 default:** in-memory + signed cookie — **no DB on day 7**.
