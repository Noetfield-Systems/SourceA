## Thesis

**Terminal-first agent products optimize for power users who live in shell/IDE.** SourceA inverts the stack: **founder sovereignty = hub clicks only** — agents run Terminal; ASF never copies `curl`, `bash`, or `python3` from chat. This is a **competitive moat** (founder-sovereign ops) and an **incident class** when hub copy regresses to shell instructions (sa-0856).

## Why hub-only wins (moat, not UX preference)

1. **Founder bandwidth** — ASF is strategic authority, not shell operator (`FOUNDER_BUSY_OPERATING_MODEL`).
2. **Honest factory** — chat claims without disk proof caused INCIDENT-016/014 family; receipts require agent execution.
3. **Canada-aligned governance** — read-only default + bounded autonomy; founder shell = unbounded risk.
4. **Multi-agent fleet** — 8 agents + Worker/Brain lanes need orchestrated shell, not founder copy-paste.
5. **Commercial integrity** — hub-only extends to no founder email/call (`FOUNDER_AGENTIC_COMMERCIAL` §3).
