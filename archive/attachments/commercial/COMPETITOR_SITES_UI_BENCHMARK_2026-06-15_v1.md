## Scoring method

Each site scored **1–10** on eight UI dimensions (higher = more professional / more effective for enterprise outbound). **Overall** = unweighted average × 10 → **0–100 score** + letter grade.

| Dimension | What it measures |
|-----------|------------------|
| **Hero** | One-glance value prop · buyer line · above-fold clarity |
| **Visual** | Typography · spacing · color · motion · polish |
| **Trust** | Logos · analyst badges · testimonials · cited stats |
| **CTA** | Demo / assess / apply paths · friction · persistence |
| **Pricing** | Transparent bands · tier cards · deposit/refund visible |
| **IA** | Information architecture — scannable vs overwhelming |
| **Proof** | Live demo · proof chain · product screenshots · replay |
| **Buyer fit** | CISO/GRC/procurement vs dev-only vs SEO blog |

**Grades:** A (85–100) · A− (80–84) · B+ (75–79) · B (70–74) · B− (65–69) · C+ (60–64) · C (&lt;60)

**Review date:** 2026-06-15 · Sources: live page fetch + prior landscape research. OSS/GitHub surfaces scored on README/docs UI only.

## Pattern checklist (apply to outbound HTML + future site refresh)

1. **Hero** — one sentence value prop · buyer line · primary CTA above fold  
2. **Stats strip** — 3 numbers max (market hypothesis or proof metrics)  
3. **Proof chain** — visual beats: request → policy → decision → receipt → replay → tamper-FAIL  
4. **Pricing cards** — 2–3 tiers · featured middle tier · deposit / refund clause visible  
5. **Trust** — shadow mode · metadata-only · fail-closed export — no certification overclaim  
6. **CTA footer** — email + live demo link · 15-min ask  
7. **Print** — `@media print` · single-column · no nav chrome
