## One sentence

> **WitnessBC** runs **human accountability workflows** (source → review → publish → correct) with **SourceA signed receipts** on every irreversible step — so teams can prove what was checked, who approved, and what changed after publish.

---

**The problem.** Teams publish reports, tips, toolkits, or public updates using agents — but agents skip sourcing checks, leak identifying detail, or “silent edit” after go-live. Spreadsheets and chat threads are not a corrections log. You need **repeatable editorial discipline** *and* **machine proof** that the discipline ran.

**What this workflow delivers.** One production loop wired to the same spine as Noetfield and AgentField:

| Layer | Role |
|-------|------|
| **WitnessBC** | Human SOP — principles, red lines, de-ID checklist, corrections protocol (existing Pro PDF toolkits) |
| **AgentField** | Agent triage, draft assembly, checklist automation, delivery |
| **SourceA** | Policy at dispatch · signed receipt · replay · tamper-evidence on every gate |

**The proof — in under five minutes.** Same chain as NW1:

> **request → policy evaluation → decision → enforcement → signed receipt → replay → tamper-FAIL → signed audit chain**

Show a **blocked publish** (red-line FAIL) · an **approved publish** · a **logged correction** · **tamper FAIL** on a hand-edited receipt.

## Reference workflow (v1 scope)

```text
INTAKE          TRIAGE           REVIEW            PUBLISH           CORRECT
(tip / draft) → (agent classify) → (founder tap) → (send / ship) → (log + link)
     │                │                 │                │                │
     └────────────────┴─────────────────┴────────────────┴────────────────┘
                              signed receipt per transition
```

| Step | Agent | Human | Receipt type | WitnessBC toolkit |
|------|-------|-------|--------------|-------------------|
| **1 Intake** | Tag topic · initial de-ID scan | — | `wbc.intake.received` | Privacy & De-identification (free/Pro) |
| **2 Triage** | Sourcing checklist · red-line flags | — | `wbc.triage.complete` | Sourcing Discipline Pack |
| **3 Review** | Draft summary + risk note | **Approve / reject** | `wbc.review.approved` | Accountability Story Template |
| **4 Publish** | Format · attach source log | Tap if sensitive | `wbc.publish.sent` | Public Record Checklist |
| **5 Correct** | Propose correction block | **Approve correction** | `wbc.correction.logged` | Corrections & Update Protocol |

**Irreversible actions (human gate required):** external publish · tip acknowledgment · paid toolkit delivery · any text naming individuals or employers (site red lines).

## Principles wired into policy (from witnessbc.com/principles)

- **Sourcing discipline** — block publish if primary source log empty for factual claims.
- **Red lines** — agent FAIL at dispatch on naming/identifiers/doxxing patterns (human override logged).
- **De-ID checklist** — four questions must PASS or escalate to review.
- **Corrections always** — material edits require new receipt; link from public correction log.
- **Independence** — no automated “pay-to-publish”; tips ≠ editorial approval.
