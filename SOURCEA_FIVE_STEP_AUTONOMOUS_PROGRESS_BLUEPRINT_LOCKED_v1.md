### Step 1 — SCAN (machine finds problems)

**Who:** Maintainer · Brain (triage only) · scripts  
**Input:** Disk only — not chat memory  
**Output:** Scan card (max 10 lines)

| Plane | Scan source |
|-------|-------------|
| LAW | authority index · READ_CHAIN · MOVED stubs |
| MACHINE | `factory-now` · `~/.sina/*.json` · validators |
| PROJECTION | hub `built_at` · monitor · dual-pick |

**Fast command:**

```bash
cd ~/Desktop/SourceA/scripts
python3 five_step_progress_machine_v1.py scan --json
```

**Scan card template:**

```markdown
## SCAN
- Pain: … (cite incident / validator)
- Goal: … (cite GOAL_HIERARCHY / PROGRAM_PROGRESS)
- Red: … (CRITICAL / FAIL / drift)
- Green: … (PASS lines worth keeping)
```

---

### Step 2 — SAY (talk to founder & developer clearly)

**Who:** Any agent facing ASF or Maintainer  
**Output:** One card per issue or fork

| Field | Rule |
|-------|------|
| **Subject** | 3–8 words |
| **Question** | One sentence |
| **In the repository** | Evidence path |
| **Effect** | If we do X → what changes (mandatory) |

**Formats (pick one per fork):**

| Type | Use |
|------|-----|
| **4 options** | A/B/C/D + Effect each |
| **Yes/No quad** | YES · NO · DEFER · DISCUSS |
| **Text box** | Conflicts · sa pick · custom rubric |

**Forbidden:** “Confirm recommended” · jargon without translation · chat-only diagnosis

**Developer = Maintainer** in this system — same SAY cards, executor runs PROVE/SHIP.

---

### Step 3 — PICK (founder decides — machine waits)

**Who:** Founder (ASF) only for law · P0 · commercial · conflict winners  
**Input:** SAY cards · Canvas live form  
**Output:** ASF order

```text
ASF: FIVE-STEP — PICK:
1. [id] KEY — short label
   Effect: one line you accept
```

**Batch 2 canonical prefix:** `ASF: FIVE-STEP —` only. Legacy `ASF: FORK-MACHINE —` and `ASF: SYS-INTEGRITY —` = same meaning, do not use in new orders. See `SOURCEA_INTEGRITY_STACK_UNIFIED_BLUEPRINT_BATCH_2_LOCKED_v1.md` §6 P1 · §10.

**Machine waits** until PICK exists for every **F** fork. Maintainer may SCAN/SAY/PROVE/SHIP on **M** rows without PICK.

---

### Step 4 — PROVE (evidence — not opinion)

**Who:** Maintainer · Worker (one sa)  
**Rule:** Done = validator PASS or receipt file path — `RESULT_DRIVEN` + INCIDENT-006

| Depth | Command | When |
|-------|---------|------|
| Fast | targeted validator | default |
| Safety | `validate-ecosystem-safety-v1.sh` | after hub/law touch |
| Critical | `find_critical_bugs.py` | before closeout |
| Deep | 100-step phase validators | quarterly audit |

**Output:** Proof lines (copy-paste paths)

---

### Step 5 — SHIP (result-driven progress)

**Who:** Maintainer ships · Founder spot-checks hub  
**Output:** Mandatory closeout

```markdown
## SHIP — session closeout
| Step | Done | Proof |
|------|------|-------|
| SCAN | ✓ | … |
| SAY | ✓ | N forks presented |
| PICK | ✓/wait | ASF lines |
| PROVE | ✓ | validators |
| SHIP | ✓ | files + Track |

### Golden insight (3 lines max)
- **Problem:** …
- **Fixed:** …
- **Your next:** 1. … 2. … 3. …

### Promote to Layer A? (if knowledge stable)
- Y/N · path · draft/active
```

**Receipt:** `~/.sina/five-step-progress-receipt-v1.json`
